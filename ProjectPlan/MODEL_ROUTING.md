# 模型能力目录与任务分工

> 目标：用统一能力标签描述不同服务商的模型，并按任务能力与成本选择合适模型。

## 一、模型能力标签

`model_providers.models_json` 从字符串数组升级为对象数组，同时兼容旧数据：

```json
[
  {
    "id": "gpt-4o-mini",
    "capabilities": {
      "input_modalities": ["text", "image"],
      "output_modalities": ["text"],
      "embedding": false
    }
  }
]
```

- 输入模态：`text`、`image`。
- 输出模态：`text`、`image`。图像输出为后续 GPT Image、Imagen 等图像生成调用预留。
- `embedding` 是独立能力，不与文字/图像生成能力混为一类。
- 旧的 `['gpt-4o', 'text-embedding-3-small']` 在读取时自动推断标签，无需数据库迁移。

能力来源按可信度排序：服务商接口返回的模态/能力元数据 → Ollama `/api/show` → 模型名称规则推断 → 用户手动修正。
服务商接口没有统一能力标准，因此推断结果不能当作绝对真相，设置界面必须允许逐模型编辑。

## 二、全局模型分工

配置以 JSON 保存到 `settings['models:role_assignments']`：

| 分工 | 能力要求 | 当前消费者 |
|---|---|---|
| 主模型 | 文本输入 + 文本输出 | 会话和角色均未指定模型时兜底 |
| 图片处理模型 | 图像输入 + 文本输出 | 已预留下发配置；图片/OCR 管线后续接入 |
| 对话总结模型 | 文本输入 + 文本输出 | 已接入滚动对话摘要 |
| 记忆更新模型 | 文本输入 + 文本输出 | 已接入现有记忆抽取器 |
| 语音理解模型 | 文本输出 | 已预留下发配置；音频输入标签随语音管线补充 |
| 快速模型 | 文本输入 + 文本输出 | 已预留下发配置；供划线翻译/搜索/解释功能使用 |
| 嵌入模型 | `embedding=true` | 已接入本地结构化记忆索引与语义检索 |

模型引用统一为 `provider_id/model_id`。保存分工时由 Rust 校验服务商、模型及能力标签；删除服务商、移除模型或取消分工所需的能力标签时，关联分工自动清空，避免悬空或能力不匹配的引用。用户显式编辑的标签始终优先，包括将全部标签清空。

## 三、主模型优先级

一次会话运行的主模型解析顺序：

1. 会话输入框选择的模型；
2. 角色卡默认模型；
3. 全局模型分工中的主模型；
4. 最终兼容回退 `gpt-4o`。

摘要、记忆更新等文本任务模型未指定时回退到本次运行的主模型。嵌入模型是例外：留空表示关闭向量索引和语义检索，不允许回退到主聊天模型。每个已配置任务都包含 provider、API Base、API Key、LiteLLM 模型名和稳定 `provider_id/model_id` 引用，通过 `taskLlmConfigs` 一次性下发给 Python sidecar。

嵌入模型配置后，Python 通过 LiteLLM 生成向量，Rust 根据响应维度延迟创建
`vec_embeddings_{dims}` cosine 索引。每次 Agent 对话前会按模型引用与正文哈希批量回填
失效记录；切换模型不需要手动迁移。嵌入请求失败时仅停用本轮向量能力，记忆工具继续使用
字符串检索。

## 四、与 Auto 权限模式的关系

Auto 不额外调用审批模型。当前主模型通过是否发起工具调用表达普通操作的执行意图；Low/Medium 自动执行，High 始终要求用户二次确认。
这样既符合“由用户所选模型决定是否执行”，也避免每个工具调用再增加一次模型请求的成本与延迟。

## 五、后续能力

- 图片消息进入上下文前，使用图片处理模型生成自然语言描述或 OCR 文本。
- 语音输入转文本后再进入普通消息链路，并新增 `audio` 输入能力标签。
- 划线悬浮窗只调用快速模型，不污染主会话历史。
- 图像生成调用落地后启用 `output_modalities=['image']` 的模型。
- 扩展图片、文档 chunk 等索引对象时，复用动态维度表和 `embedding_items` 元数据，并将 partition key 抽象为 namespace type/id。原始向量行不进 D1；大型 RAG 可按 `STORAGE_AND_RAG.md` 导出客户端加密的便携制品到 R2/Google Drive。
