# agnes-agent UI/UX 设计规范与界面方案 (现代极简版)

本文件定义了 `agnes-agent` 桌面客户端 (Tauri + React) 的极简、现代化设计方案。本设计基于「对话与执行聚焦」的理念，将日常不常用的配置模块（角色切换、人设编辑、记忆修改、同步等）收纳进统一设置中心，保持主界面极其纯粹干净。

---

## 1. 设计理念 (Design Philosophy)

*   **现代极简主义 (Modern Minimalism)**：采用干净、开阔的 Zinc/Slate 灰白配色系统，强调信息层级与大留白。支持系统级的明暗主题切换，不堆砌繁复的霓虹、赛博或多色渐变，回归纯净高对比度的文本阅读体验。
*   **功能聚焦 (Feature Focus)**：主界面专注于**对话 (Chat)** 与 **工具审计/人类审批 (Audit & Approval)**，去除所有干扰元素（如主界面常驻的角色切换列表和记忆面板）。
*   **渐进式暴露 (Progressive Disclosure)**：非高频使用的管理交互（如添加角色、编辑角色人设、直接手写记忆文件 USER.md/MEMORY.md）全部移入统一配置后台，随用随开。
*   **直观的执行反馈 (Clear Execution Pipeline)**：保留可收缩的 AI 思考轨迹与高度醒目的工具审批卡片，确保 Agent 执行本地命令的透明度与控制力。

---

## 2. 界面布局架构 (Interface Architecture)

应用采用两栏布局（左侧可折叠边栏 + 右侧主对话区），极具亲和力，类似现代高水准的知识库或轻量对话工具。

```text
+--------------------------------------------------------------------------------------------------------+
|                                           TOP TITLEBAR (Window Controls & Cloud Sync Status)           |
+--------------------------------------------------------------------------------------------------------+
| COLLAPSIBLE SIDEBAR       |  CHAT WORKSPACE (Focus Area)                                               |
|                           |                                                                            |
|  +---------------------+  |  +----------------------------------------------------------------------+  |
|  | Agnes (Active Agent)|  |  | [Toggle]  Session: 前端 UI 界面设计规划                                  |  |
|  | Maid Code Helper    |  |  +----------------------------------------------------------------------+  |
|  +---------------------+  |  |                                                                      |  |
|  |                     |  |  | [User Message]                                                       |  |
|  | SESSIONS            |  |  |   看一下当前项目的文件修改状态。                                     |  |
|  |                     |  |  |                                                                      |  |
|  | +-[Sess 1: UI Dev]  |  |  | [Agent Message]                                                      |  |
|  | +-[Sess 2: DB Mig]  |  |  |   +-[Thought: collapsable thinking flow]---------------------------+ |  |
|  |                     |  |  |   +----------------------------------------------------------------+ |  |
|  |                     |  |  |                                                                      |  |
|  |                     |  |  |   +-[Tool Request: Approval Card]----------------------------------+ |  |
|  |                     |  |  |   | Run: git status        [Approve (Ctrl+Enter)]   [Reject (Esc)] | |  |
|  |                     |  |  |   +----------------------------------------------------------------+ |  |
|  |                     |  |  |                                                                      |  |
|  |                     |  |  |   +-[Tool Output: terminal log]------------------------------------+ |  |
|  |                     |  |  |   | $ git status --short                                           | |  |
|  |                     |  |  |   +----------------------------------------------------------------+ |  |
|  |                     |  |  |                                                                      |  |
|  |                     |  |  |   这里是命令执行结果...接下来我们...                                 |  |
|  |                     |  |  +----------------------------------------------------------------------+  |
|  |                     |  |  | Chat Input Area (Textarea + Send)                                    |  |
|  +---------------------+  |  +----------------------------------------------------------------------+  |
|  | [Settings Icon]     |  |                                                                            |
|  +---------------------+  +----------------------------------------------------------------------------+
```

### 2.1 左侧可折叠边栏 (Collapsible Sidebar)
*   **当前 Agent 信息卡片**：
    *   展示当前活跃的 Agent 头像（现代纯色圆形头像）、名字、个性标签。只做信息展示，不做点击切换。
*   **会话管理 (Sessions)**：
    *   顶部提供 `+ 新建会话` 按钮。
    *   列表展示当前 Agent 下关联的历史会话列表，支持点击切换、重命名与删除会话。
*   **底部控制区**：
    *   **设置入口**（齿轮图标）：点击拉起覆盖全屏的「配置与管理中心」大弹窗。

### 2.2 主对话工作区 (Chat Workspace)
*   **Header 控制栏**：
    *   侧边栏折叠/展开开关。
    *   当前会话标题与已用 Context 窗口 Token 计量。
*   **消息对话流**：
    *   **用户消息**：现代气泡或直接文本块样式。
    *   **Agent 消息**：
        *   **思维卡片 (Thought)**：极简的灰色左边线，默认收缩，点击可展开完整的 Agent 思维链。
        *   **工具审批卡 (Tool Approval)**：根据工具风险，显示干净的黄边（中危）或红边（高危）卡片，内嵌指令详情，提供直观的「同意」和「拒绝」按钮。
        *   **终端输出 (Terminal Log)**：黑色等宽字体的只读折叠块，呈现本地命令运行 stdout 实时状态。
        *   **正文输出**：标准、优雅的 Markdown 渲染。
*   **输入交互框**：
    *   无边框或细边框设计的文本域，支持 Enter 发送，提供当前模型的快速切线。

## 3. 统一设置中心 (Settings & Management Dashboard)

点击设置齿轮后，拉起该全屏覆盖层。这是管理 `agnes-agent` 核心数据的控制台。

```text
+------------------------------------------------------------------------------------+
|  设置与控制中心                                                           [关闭 X] |
+------------------------------------------------------------------------------------+
|  左侧菜单                 |  右侧配置详情区域                                      |
|                           |                                                        |
|  1. 角色管理 (Agents)     |  [ 角色列表 ]   [+ 新建角色]                           |
|     - 选择/添加/删除      |  - Agnes (当前活跃)  [编辑人设] [配置工具]             |
|     - 编辑人设与提示词    |  - Nova              [编辑人设] [配置工具]             |
|                           |  - Bard              [编辑人设] [配置工具]             |
|  2. 记忆编辑器 (Memory)   |                                                        |
|     - 编辑 USER.md        |  [ 记忆管理: Agnes ]                                   |
|     - 编辑 MEMORY.md      |  - 每次必注入文本编辑器 (Markdown)                      |
|     - 检索/清理向量库     |  - 语义向量记忆条目列表 (支持检索与一键删除)            |
|                           |                                                        |
|  3. 模型与密钥 (LLM API)  |  [ LiteLLM 密钥配置 ]                                  |
|                           |  - API Key: •••••••••••                                |
|  4. 云端同步 (Sync)       |                                                        |
|                           |  [ Cloudflare D1 Sync ]                                |
|  5. 工具审计日志 (Audit)   |  - 同步网关 URL 与离线策略设定                         |
+------------------------------------------------------------------------------------+
```

### 3.1 页面 1：角色卡管理 (Agent Manager)
*   **智能体列表切换**：用户在此切换当前正在对话的活跃角色（设为 Active）。提供「删除角色」与「添加新角色」功能。
*   **人设与指令卡片编辑**：
    *   修改姓名、短描述、默认模型。
    *   编辑 `Persona (性格设定)` 与 `System Prompt (指令前缀)`。
    *   **工具授权管理 (Tool Policies)**：配置此角色可调用的工具（Shell、File、Git），以及每个工具的审批策略（如每次执行都要询问、仅在涉及写动作时询问、自动放行）。

### 3.2 页面 2：记忆三维编辑器 (Memory Inspector)
*   **USER.md / MEMORY.md 编辑器**：
    *   选择对应的角色后，展示其关联的 markdown 记忆文件。提供富文本/Raw 格式切换，用户修改后直接提交存回 DB，并自动回写本地文件。
*   **语义记忆库清理 (Semantic Database)**：
    *   列表列出所有被抽取沉淀的语义事实，提供置信度评分和来源标注。用户可一键剔除 AI 产生的幻觉。

### 3.3 页面 3：模型与云同步设置
*   管理 API Key（存储于系统 Keyring）。
*   配置 Cloudflare Workers 增量同步参数，查看同步冲突解决日志。

## 4. 颜色与视觉规范 (Visual System Tokens)

从 Cyber 风格切换为**现代质感简洁风**，设计规范如下：

### 4.1 调色板 (Color Palette)

| 颜色名称 | 变量/Hex | 使用场景 |
|---|---|---|
| **Background (主背景)** | `bg-zinc-50` / `bg-zinc-950` | 页面最底层背景，干净爽朗 |
| **Sidebar (侧边栏背景)** | `bg-zinc-100/50` / `bg-zinc-900/60` | 与主聊天区产生视觉区隔 |
| **Card / Bubbles (卡片)** | `bg-white` / `bg-zinc-900` | 消息气泡、输入框、控制面板 |
| **Border (细分割线)** | `border-zinc-200` / `border-zinc-800` | 极其纤细的 1px 描边 |
| **Accent Violet (主题紫)**| `bg-violet-600` / `text-violet-600` | 按钮激活态、首选模型指示、思维指示 |
| **Text Primary (正文)** | `text-zinc-900` / `text-zinc-50` | 正文字体，强调清晰可读 |
| **Text Secondary (次要)** | `text-zinc-500` / `text-zinc-400` | 时间戳、备注、折叠描述 |

### 4.2 细节规范
*   **圆角系统 (Border Radius)**：统一采用较大而圆润的现代圆角设计（`rounded-2xl` 适用于主气泡/卡片，`rounded-xl` 适用于按钮/输入框）。
*   **阴影系统 (Shadows)**：使用现代柔和的扩散阴影（`shadow-sm` 用于卡片，`shadow-xl` 用于浮动弹窗），去除扁平化带来的简陋感。
*   **排版**：正文字号略微调大，行间距设计为 `leading-relaxed`，极佳的阅读舒适感。
