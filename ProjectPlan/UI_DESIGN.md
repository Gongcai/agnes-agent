# agnes-agent UI/UX 设计规范与界面方案 (Claude Desktop 风格)

本文件定义了 `agnes-agent` 桌面客户端 (Tauri + React) 的极简、现代化设计方案。主界面是聊天、记忆、知识库、网盘、日历和待办的统一工作台；角色人设、Provider 授权、同步和其他低频配置收纳到设置中心。

---

## 1. 设计理念 (Design Philosophy)

*   **温暖的人文工具感 (Warm Humanist Utility)**：主要参考 `claude-desktop-reconstruction.html` 的 Claude Code Desktop 产品界面，以暖纸色侧栏、白色工作区、陶土色动作和深色代码面建立层级；`DESIGN-claude.md` 仅补充品牌色与排版原则。界面保持桌面工具的信息密度，不照搬营销页的大字号与卡片分区。
*   **功能聚焦 (Feature Focus)**：主界面使用稳定的子功能导航，聊天、记忆、知识库、网盘、日历和待办分别进入独立视图。当前视图内只保留与当前任务相关的操作。
*   **渐进式暴露 (Progressive Disclosure)**：非高频使用的管理交互（如添加角色、编辑角色人设、直接手写记忆文件 USER.md/MEMORY.md）全部移入统一配置后台，随用随开。
*   **直观的执行反馈 (Clear Execution Pipeline)**：保留可收缩的 AI 思考轨迹与高度醒目的工具审批卡片，确保 Agent 执行本地命令的透明度与控制力。

---

## 2. 界面布局架构 (Interface Architecture)

应用采用两栏布局（左侧可折叠边栏 + 右侧子功能内容区）。左侧负责功能导航与上下文选择，右侧根据当前路由显示聊天、知识库、网盘、日历或待办。

```text
+--------------------------------------------------------------------------------------------------------+
|                                           TOP TITLEBAR (Window Controls & Cloud Sync Status)           |
+--------------------------------------------------------------------------------------------------------+
| COLLAPSIBLE SIDEBAR       |  ACTIVE FEATURE VIEW                                                      |
|                           |                                                                            |
|  Agnes / Active Agent     |  +----------------------------------------------------------------------+  |
|                           |  | [Toggle]  Chat / Knowledge / Drive / Calendar / Tasks                  |  |
|  FEATURES                 |  +----------------------------------------------------------------------+  |
|    Chat                   |  |                                                                      |  |
|    Memory                 |  |  Current feature content                                              |  |
|    Knowledge              |  |                                                                      |  |
|    Drives                 |  |  Chat: message stream + tool approvals                                |  |
|    Calendar               |  |  Knowledge: document/index status                                    |  |
|    Tasks                  |  |  Drives: provider/files/replicas                                      |  |
|                           |  |  Calendar: calendar views                                             |  |
|  CHAT SESSIONS       [v]  |  |  Tasks: task lists                                                    |  |
|    Session A              |  |                                                                      |  |
|    Session B              |  |                                                                      |  |
|                           |  |                                                                      |  |
|  WORKSPACE SESSIONS  [v]  |  |                                                                      |  |
|    Project Alpha          |  |                                                                      |  |
|      Session C            |  |                                                                      |  |
|                           |  +----------------------------------------------------------------------+  |
|  [Sync] [Settings]     |  |                                                                            |
+------------------------+--+----------------------------------------------------------------------------+
```

### 2.1 左侧可折叠边栏 (Collapsible Sidebar)
*   **当前 Agent 信息卡片**：
    *   展示当前活跃的 Agent 头像（现代纯色圆形头像）、名字、个性标签。只做信息展示，不做点击切换。
*   **子功能列表 (Features)**：
    *   使用图标 + 文本导航到聊天、记忆、知识库、网盘、日历和待办。
    *   当前选中项使用安静的背景/左边线状态，不使用大卡片或营销式导航块。
    *   未实现功能在开发期不显示空页入口，随子功能完成逐个开放。
*   **聊天会话 (Chat Sessions)**：
    *   标题行提供折叠/展开和新建命令，折叠时不渲染列表。
    *   只展示当前 Agent 下未绑定 workspace 的普通会话，支持切换、重命名、置顶与删除。
*   **工作区会话 (Workspace Sessions)**：
    *   与普通聊天会话独立折叠，按逻辑 workspace 分组显示关联会话。
    *   workspace 未在本机绑定路径时仍可进入会话，但组标题显示未绑定状态，文件工具不可执行。
*   **折叠状态**：
    *   子功能列表保持可见；两类会话列表可独立折叠。
    *   侧边栏整体折叠时转为固定宽度图标轨，不显示会话文字，展开后尺寸稳定不跳动。
    *   两类列表的折叠偏好只保存在本机 UI settings，不参与 D1 同步冲突。
*   **底部控制区**：
    *   显示同步状态图标和**设置入口**（齿轮图标）；设置仍打开统一配置与管理中心。

### 2.2 子功能内容区 (Feature Views)
*   **聊天**：对话流、工具审批和输入区。
*   **记忆**：当前 Agent 的 USER.md / MEMORY.md / 结构化记忆，编辑器与检索列表分视图呈现。
*   **知识库**：collection、源文件、解析/分块/向量进度、设备索引覆盖和检索测试。
*   **网盘**：Provider 账户、文件浏览、对象副本、同步队列和授权错误。
*   **日历**：月/周/日/议程视图；多日历按颜色叠加和显隐，待办期限作为独立图层；选中日期后在下方显示当天议程。事件通过结构化弹窗编辑标题、日期/时间、时区、全天和常用重复规则。
*   **待办**：我的一天、重要、已计划、全部、已完成五类智能视图与自定义列表；快速添加、完成分组和任务详情抽屉覆盖重要、我的一天、日期/精确时间、时区、重复、步骤、备注与删除。
*   每个视图都使用自己的工具栏、空状态和错误状态，不将整个子功能包在浮动卡片中。

#### 日历与待办交互约束

*   “我的一天”必须由用户手动加入，且仅对当天有效；从该智能视图快速添加时默认加入当天。
*   日期任务在日历中显示为全天项，精确时间任务显示在时间槽；点击任务图层应切换到待办并打开对应详情。
*   重复任务完成后保留完成实例并显示下一实例，重新打开时由数据层协调自动实例，前端不自行推算 RRULE。
*   普通编辑器不显示裸 ISO 8601 或 RRULE 输入；常用重复规则使用选项菜单，自定义规则只允许保留已有值。
*   提醒不归属于单个页面。顶部通知中心统一承载 AI 回复、AI 许可请求、任务到期和事件开始；点击通知跳转到相应聊天、任务详情或日历事件。系统原生通知以后作为可替换投递通道接入。

### 2.3 主对话工作区 (Chat Workspace)
*   **Header 控制栏**：
    *   侧边栏折叠/展开开关。
    *   当前会话标题与已用 Context 窗口 Token 计量。
*   **消息对话流**：
    *   **用户消息**：现代气泡或直接文本块样式。
    *   **Agent 消息**：
        *   **思维卡片 (Thought)**：极简的灰色左边线，默认收缩，点击可展开完整的 Agent 思维链。
        *   **工具审批卡 (Tool Approval)**：根据工具风险，显示干净的黄边（中危）或红边（高危）卡片，内嵌指令详情，提供直观的「同意」和「拒绝」按钮。
        *   **终端输出 (Terminal Log)**：黑色等宽字体的只读折叠块，呈现本地命令运行 stdout 实时状态。
        *   **正文输出**：标准、优雅的 Markdown 渲染。
*   **输入交互框**：
    *   无边框或细边框设计的文本域，支持 Enter 发送，提供当前模型的快速切线。

## 3. 统一设置中心 (Settings & Management Dashboard)

点击设置齿轮后，拉起该全屏覆盖层。这是管理 `agnes-agent` 核心数据的控制台。

```text
+------------------------------------------------------------------------------------+
|  设置与控制中心                                                           [关闭 X] |
+------------------------------------------------------------------------------------+
|  左侧菜单                 |  右侧配置详情区域                                      |
|                           |                                                        |
|  1. 角色管理 (Agents)     |  [ 角色列表 ]   [+ 新建角色]                           |
|     - 选择/添加/删除      |  - Agnes (当前活跃)  [编辑人设] [配置工具]             |
|     - 编辑人设与提示词    |  - Nova              [编辑人设] [配置工具]             |
|                           |  - Bard              [编辑人设] [配置工具]             |
|  2. 记忆编辑器 (Memory)   |                                                        |
|     - 编辑 USER.md        |  [ 记忆管理: Agnes ]                                   |
|     - 编辑 MEMORY.md      |  - 每次必注入文本编辑器 (Markdown)                      |
|     - 检索/清理向量库     |  - 语义向量记忆条目列表 (支持检索与一键删除)            |
|                           |                                                        |
|  3. 模型与密钥 (LLM API)  |  [ LiteLLM 密钥配置 ]                                  |
|                           |  - API Key: •••••••••••                                |
|  4. 同步与外部服务       |  [ Cloudflare D1 / R2 / Google OAuth ]                  |
|     - 同步凭证          |  - 网关、设备、网盘/日历/Task Provider 授权          |
|  5. 工具审计日志 (Audit)   |  - 同步错误、副本与本地缓存策略                       |
+------------------------------------------------------------------------------------+
```

### 3.1 页面 1：角色卡管理 (Agent Manager)
*   **智能体列表切换**：用户在此切换当前正在对话的活跃角色（设为 Active）。提供「删除角色」与「添加新角色」功能。
*   **人设与指令卡片编辑**：
    *   修改姓名、短描述、默认模型。
    *   编辑 `Persona (性格设定)` 与 `System Prompt (指令前缀)`。
    *   **工具授权管理 (Tool Policies)**：配置此角色可调用的工具（Shell、File、Git），以及每个工具的审批策略（如每次执行都要询问、仅在涉及写动作时询问、自动放行）。

### 3.2 页面 2：记忆高级维护 (Memory Inspector)
主界面“记忆”子功能承担日常查看、搜索和编辑；设置中心仅保留按 Agent 切换、批量清理、索引重建和原始 Markdown 维护等低频管理功能。
*   **USER.md / MEMORY.md 编辑器**：
    *   选择对应的角色后，展示其关联的 markdown 记忆文件。提供富文本/Raw 格式切换，用户修改后直接提交存回 DB，并自动回写本地文件。
*   **语义记忆库清理 (Semantic Database)**：
    *   列表列出所有被抽取沉淀的语义事实，提供置信度评分和来源标注。用户可一键剔除 AI 产生的幻觉。

### 3.3 页面 3：模型与云同步设置
*   管理 API Key（存储于系统 Keyring）。
*   配置 Cloudflare Workers 增量同步参数，查看同步冲突解决日志。

## 4. 颜色与视觉规范 (Visual System Tokens)

视觉系统采用 Claude Desktop 的暖纸色与暖黑文字，以**陶土色 (Clay / Coral)** 作为唯一品牌动作色；工具输出和代码使用暖黑深色面。颜色真理源位于 `src/index.css` 的 `--claude-*` tokens，旧的绿色 utility 仅作为迁移输入，不再定义最终视觉。

### 4.1 调色板 (Color Palette)

| 颜色名称 | 变量/Hex | 使用场景 |
|---|---|---|
| **Canvas (主背景)** | `--claude-canvas` / `#FAF9F5` | 输入区、工具条、暖纸色空白 |
| **Sidebar (侧边栏)** | `--claude-sidebar` / `#F5F3ED` | 功能和会话导航，与白色工作区形成一级层差 |
| **Surface (主工作区)** | `--claude-surface` / `#FFFFFF` | 聊天正文、设置内容、日历和列表工作面 |
| **Surface Soft** | `--claude-surface-soft` / `#F0EDE6` | 用户消息、安静选中态、次级控件 |
| **Border** | `--claude-border` / warm black 13% | 0.5-1px 分隔线与控件描边 |
| **Clay (品牌动作)** | `--claude-clay` / `#D97757` | 主按钮、当前项边线、焦点和 Agent 状态 |
| **Ink (主文字)** | `--claude-ink` / `#1F1E1B` | 标题、正文和主要图标 |
| **Muted (次要文字)** | `--claude-muted` / `#77736A` | 时间戳、Token、状态说明 |
| **Dark Surface** | `--claude-dark` / `#262624` | 终端、代码与高对比工具输出 |

### 4.2 细节规范
*   **圆角系统 (Border Radius)**：普通按钮、列表项和卡片不超过 8px；聊天输入框使用 Claude Desktop 标志性的 16px，用户气泡使用 18px 并保留一个 5px 尾角。
*   **阴影系统 (Shadows)**：深度主要依靠表面色差和细边框；常驻面板不使用阴影，仅弹窗和上下文菜单使用克制的 16-24px 扩散阴影。
*   **排版**：界面统一使用系统可用的 `Noto Serif` 英文 serif，并以 `Noto Serif SC / Noto Serif CJK SC` 作为中文回退；代码和 Token 元数据使用 `JetBrains Mono` 等宽栈。所有文字 letter-spacing 为 0。

## 5. 未来主题自定义扩展 (Future Theme Customization Roadmap)

为支持日后用户深度自定义主色系与副色系，本项目的视觉设计已前置完成了「主题解耦」：

1.  **CSS 变量驱动**：
    *   将 `src/index.css` 中的 `--background`、`--foreground`、`--primary` 等变量作为颜色真理源。
    *   在 Tailwind 样式中统一使用这些变量对应的 utility 类（如 `bg-background`、`text-primary`），避免硬编码特定颜色类名。
2.  **动态样式注入**：
    *   在 V0.4 中，用户的自定义配色将作为 JSON 存入本地 SQLite 的 `settings` 表中。
    *   前端 React 启动时，通过自定义 hook 读取该配置，并通过 JS 动态修改 HTML `document.documentElement.style` 的 CSS 变量值（如覆盖 `--claude-clay`），实现实时、全局且无刷新的主题切换。
